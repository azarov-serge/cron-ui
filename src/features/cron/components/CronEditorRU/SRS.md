# SRS: CronEditor и CronPage

**Версия:** 1.6  
**Статус:** сверено с реализацией  
**Компонент:** `src/features/cron/components/CronEditor/CronEditor.tsx`  
**Страница:** `src/features/cron/pages/CronPage.tsx` (экспорт `CronPage`)  
**TimePicker:** `src/shared/components/TimePicker/`

---

## 1. Назначение

**`CronEditor`** — форма редактирования расписания в стиле SQL Server Agent (SSMS). На входе и выходе — пятичастное cron-выражение (без секунд). Компонент не привязан к конкретному API: интеграция с бэкендом — задача вызывающего кода.

**`CronPage`** — демо и песочница: три вкладки, синхронизация `cron` с URL, inline-редактор на вкладке «Конструктор», настройка `CronOptions` на отдельной вкладке.

## 2. Контракт с бэкендом

### 2.1. Что приходит с бэкенда

Расписание — **объект cron в snake_case** или **строка** `минута час день_месяца месяц день_недели`.

| Поле | Тип | Описание |
|------|-----|----------|
| `minute` | `string` | 0–59, `*`, `*/N` |
| `hour` | `string` | 0–23, `*`, `*/N` |
| `day_of_month` | `string \| null` | 1–31 или `null` → `*` |
| `month_of_year` | `string \| null` | 1–12 или `null` → `*` |
| `day_of_week` | `string \| null` | 0–6, списки, `день#неделя` или `null` → `*` |

Примеры выражений:

```text
0 9 * * 1        — по понедельникам в 09:00
0 9 * * 1#1      — пн в 1-ю неделю месяца, 09:00
0 9 15 * *       — 15-го числа в 09:00
30 14 25 3 *     — однократно 25 марта в 14:30
*/10 * * * *     — каждые 10 минут
```

`CronEditor` принимает начальное значение через проп `cron`. Невалидная строка (не 5 частей) при разборе заменяется на дефолт `0 9 * * *`.

### 2.2. Рекомендуемая сущность API

```ts
type ScheduleEntity = {
  id: string;
  name: string;
  enabled: boolean;
  cron: CronResponse; // или cron_expression: string
};
```

`CronPage` **не сохраняет** `name` и `enabled` — только демонстрирует редактирование `cron`. Метаданные — задача вызывающего кода.

`CronOptions` на бэкенд **не сохраняются**.

### 2.3. Что уходит на бэкенд

При каждом изменении UI `CronEditor` вызывает `onChange(cron)`. Сериализация: `cron.toJson()` (snake_case) или `cron.toExpression()`.

```json
{
  "cron": {
    "minute": "0",
    "hour": "9",
    "day_of_week": null,
    "day_of_month": null,
    "month_of_year": null
  }
}
```

На `CronPage` `onChange` обновляет `cron` в состоянии и в URL.

## 3. CronEditor

### 3.1. Пропсы

| Свойство | Тип | Описание |
|----------|-----|----------|
| `cron` | `Cron` | Начальное расписание; без пропа — `0 9 * * *` (ежедневно 09:00 в форме) |
| `onChange` | `(cron: Cron) => void` | Вызывается при каждом изменении расписания или `options` в store |
| `options` | `CronOptions` | Ограничения UI и валидации |

Разметка — styled `Form` без `id` и без submit. Внешняя кнопка «Сохранить» не требуется.

При смене `cron` снаружи store синхронизируется через `useCronEditorSync`: `initFromCron` вызывается только если `cron.toExpression()` изменился (защита от лишних сбросов при эхо `onChange`).

### 3.2. Архитектура

| Часть | Путь | Назначение |
|-------|------|------------|
| Store | `hooks/useCronEditorStore.ts` | Zustand: `schedule`, `options`, действия, `useCronEditorSync`, `useCronEditorOnChange` |
| Модель | `models/schedule/Schedule.ts` | UI-состояние ↔ `Cron` |
| Секции | `components/` | `ScheduleTypeSection`, `OneTimeSection`, `RecurringSection`, `CronOutputSection` |
| Поля вывода | `components/CronFields/` | `CronExpressionField`, `CronDescriptionField` |

### 3.3. CronOptions

| Параметр | По умолчанию | Поведение |
|----------|--------------|-----------|
| `scheduleTypes` | `recurring`, `one-time` | Один элемент — блок «Тип расписания» скрыт |
| `occursFrequencies` | `daily`, `weekly`, `monthly` | Один элемент — список «Выполняется» скрыт |
| `dailyFrequencies` | `once`, `every` | Один элемент — радиокнопки «Ежедневная частота» скрыты |
| `minuteStep` | `1` | Шаг минут в полях времени и в «каждые N минут» |
| `weeklyWeekNumbers` | `false` | Чекбоксы недель месяца (1–5) для еженедельного режима |
| `requires` | `[]` | Обязательные поля: `weeklyWeekDays`, `weeklyWeekNumbers` |

Допустимые `minuteStep`: `1, 2, 3, 4, 5, 6, 10, 12, 15, 20, 30`.

При изменении `options` store вызывает `syncOptions` и приводит форму к разрешённым значениям. При включении недель месяца без выбора — по умолчанию **1-я неделя**.

### 3.4. Секции формы

1. **Тип расписания** — повторяющееся / однократно
2. **Однократное выполнение** — дата `DD.MM.YYYY`, время
3. **Частота** — ежедневно / еженедельно / ежемесячно  
   - дни недели; при `weeklyWeekNumbers` — недели месяца (1–5)  
   - ежемесячно — день месяца (1–31)
4. **Ежедневная частота** — один раз в (время) / каждые N минут или часов
5. **cron** — сырое выражение; **Описание** — локализованный текст

Время: `TimePickerField` из `@shared/components/TimePicker` — колонки «Часы» / «Мин.» и ввод `HH:mm` (`dimension="s"`).

### 3.5. Соответствие UI → cron

| Настройка | Выражение |
|-----------|-----------|
| Ежедневно, 09:00 | `0 9 * * *` |
| Еженедельно, пн, 09:00 | `0 9 * * 1` |
| Еженедельно, пн, 1-я неделя | `0 9 * * 1#1` |
| Ежемесячно, 15-е, 09:00 | `0 9 15 * *` |
| Каждые 10 минут | `*/10 * * * *` |
| Каждые 2 часа | `0 */2 * * *` |
| Однократно 25.03, 14:30 | `30 14 25 3 *` |

Дни недели в cron: 0 = вс, 1 = пн, …, 6 = сб.

Еженедельно с неделями месяца: нотация `день#неделя` (напр. `1#1,3#2`). Без выбранных дней → `*`; без выбранных недель при включённой опции → обычный weekly.

### 3.6. Валидация

Ошибки показываются в UI, но **не блокируют** `onChange`:

| Условие | Сообщение |
|---------|-----------|
| `requires: weeklyWeekDays`, нет дней | «Выберите хотя бы один день недели» |
| `requires: weeklyWeekNumbers`, нет недель | «Выберите хотя бы одну неделю месяца» |
| Интервал вне диапазона | «от N до M минут/часов» |

`one-time` — без проверок recurring-полей. Интервал: минуты `minuteStep`…59 (кратно шагу), часы 1…23.

### 3.7. Ограничения UI

- Нельзя собрать `* * * * *`, «каждые 100 минут», составные интервалы.
- Сложные cron при загрузке могут упроститься в представлении формы.
- Год однократного расписания в UI — текущий; в выражение не входит.

## 4. CronPage

### 4.1. Состояние

| Состояние | Источник | Назначение |
|-----------|----------|------------|
| `activeTab` | URL (`?tab=`) | `constructor` \| `editorParams` \| `checker` |
| `cron` | URL (`?cron=`) | Текущее расписание |
| `checkerExpression` | URL / локально | Строка на вкладке «Проверка cron» |
| `cronParamError` | разбор URL | Ошибка невалидного cron в адресе |
| `editorOptions` | `useState` | `CronOptions` для `CronEditor` (не в URL) |

Хук: `useCronPageSearchParams`. Метод `submitCron` обновляет `cron`, `checkerExpression` и URL.

### 4.2. Вкладки

| Вкладка | `?tab=` | Содержимое |
|---------|---------|------------|
| Конструктор cron | _(по умолчанию)_ | подсказка, inline `CronEditor` (`onChange={submitCron}`) |
| Параметры редактора | `editorParams` | панели `CronOptions` и «Обязательные поля» |
| Проверка cron | `checker` | `CronChecker`, кнопка «Редактор cron» → `submitCron` + переход на конструктор |

Модального окна нет. Кнопки «Сохранить расписание» нет — cron пишется в URL при каждом изменении полей.

### 4.3. Панель «Параметры редактора»

| UI | Поле `editorOptions` |
|----|----------------------|
| Тип расписания | `scheduleTypes` |
| Частота | `occursFrequencies` |
| Ежедневная частота | `dailyFrequencies` |
| Недели месяца | `weeklyWeekNumbers` |
| Шаг минут | `minuteStep` |

Минимум один пункт в каждой группе чекбоксов всегда выбран.

### 4.4. Панель «Обязательные поля»

| Чекбокс | `requires` |
|---------|------------|
| Дни недели обязательны | `weeklyWeekDays` |
| Недели месяца обязательны | `weeklyWeekNumbers` (disabled, если недели выключены) |

### 4.5. Использование вне демо

В продукте `CronOptions` задаётся кодом, без панелей `CronPage`:

```tsx
import {
  CronEditor,
  type CronOptions,
} from '@features/cron/components/CronEditor';
import { Cron } from '@features/cron/components/CronEditor/models/cron';

<CronEditor
  cron={task.cron}
  options={{
    scheduleTypes: ['recurring'],
    occursFrequencies: ['daily', 'weekly'],
    dailyFrequencies: ['once'],
    minuteStep: 5,
  }}
  onChange={saveCron}
/>
```

## 5. Сценарии приёмки

**CronEditor**

1. Загрузка `cron` корректно заполняет форму.
2. Любое изменение поля вызывает `onChange` с cron, совпадающим с блоком «cron» на форме.
3. Сужение `options` скрывает секции и сбрасывает недопустимые значения.
4. `weeklyWeekNumbers: true` — недели видны; при первом включении выбрана 1-я.
5. `requires` показывает сообщения об ошибках; `onChange` при этом всё равно вызывается.
6. `minuteStep` ограничивает время и интервал в минутах.
7. Смена `cron` снаружи обновляет форму без бесконечного цикла с `onChange`.

**CronPage**

8. Вкладка «Параметры редактора» меняет поведение `CronEditor` на «Конструкторе».
9. Изменения в редакторе сразу обновляют `?cron=` в URL и блок cron/описание.
10. «Проверка cron» → валидное выражение → «Редактор cron» переносит cron на вкладку конструктора.
